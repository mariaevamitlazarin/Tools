# Synthetic Data Generators Classification

Three small, dependency-light generators for producing **fake-but-learnable** datasets so you can build and test ML pipelines *before* real data exists. Each combines a scikit-learn data maker (the learnable numeric signal) with [Faker](https://faker.readthedocs.io/) (realistic metadata columns like names, emails, dates). Every generator returns a single **pandas DataFrame**.

| File | Task | sklearn maker | Target column |
|------|------|---------------|---------------|
| `synthetic_generator_classification.py` | Classification | `make_classification` | `target` (discrete) |
| `synthetic_generator_regression.py` | Regression | `make_regression` | `target` / custom (continuous) |
| `synthetic_generator_clustering.py` | Clustering | `make_blobs` / `make_moons` / `make_circles` | `cluster` (ground truth) |

## Install

```bash
pip install faker scikit-learn pandas
```

## Quick start

Each file exposes one function, `generate(...)`, returning a DataFrame.

```python
from synthetic_generator import generate            # classification
# from synthetic_generator_regression import generate
# from synthetic_generator_clustering import generate

df = generate(n_rows=2000, seed=7)
print(df.head())
```

Run any file directly (`python synthetic_generator.py`) to see a worked example printed to stdout.

## When to use this

- **Bootstrapping a pipeline** — write and test ingestion, preprocessing, training, and evaluation code while the real dataset is still being collected or cleared for use.
- **CI / unit tests** — fast, deterministic (seeded) data so tests don't depend on a large real dataset.
- **Demos & teaching** — reproducible examples with realistic-looking columns.
- **Stress testing** — deliberately inject imbalance, noise, missing values, duplicates, and outliers to see how your code copes.
- **Benchmarking models** — tune difficulty knobs to compare algorithms on easy vs. hard versions of a problem.

## When NOT to use this

- **Final model validation.** Synthetic signal is generated by a known process; good scores here do **not** guarantee real-world performance. Always validate on real data before shipping.
- **Capturing real distributions.** These generators do not learn from or mimic any real dataset's joint distribution. For that, look at SDV, CTGAN, or similar.
- **Privacy-preserving substitution.** This is invented data, not anonymized real data — but it also carries none of the statistical properties of your real data.

## Shared knobs (all three generators)

| Argument | Effect |
|----------|--------|
| `n_rows` | Number of rows. |
| `n_features` | Number of numeric `feat_*` columns. |
| `faker_fields` | Tuple of realistic columns to add. Available: `name`, `email`, `country`, `job`, `company`, `phone`, `address`, `signup_date`, `city`. |
| `correlate_with_target` / `correlate_with_cluster` | If `True`, adds an `age` column whose distribution shifts with the label — a realistic-looking field that actually carries signal. |
| `missing_frac` | Fraction of numeric cells set to `NaN`. |
| `duplicate_frac` | Fraction of rows duplicated (then shuffled in). |
| `outlier_frac` | Fraction of numeric cells perturbed with large noise. |
| `seed` | Random seed — fixes both sklearn and Faker for reproducibility. |
| `locale` | Faker locale, e.g. `"en_US"`, `"pt_BR"`, `"fr_FR"`. |

## Task-specific knobs

**Classification** (`synthetic_generator.py`)
- `n_informative`, `n_redundant` — how much of the feature space is real signal vs. linear combinations.
- `n_classes` — number of classes.
- `class_sep` — separation between classes; **higher = easier**.
- `weights` — class proportions, e.g. `[0.9, 0.1]` for imbalanced / fraud-style data.

**Regression** (`synthetic_generator_regression.py`)
- `n_informative` — number of features that actually drive the target.
- `noise` — std of gaussian noise on the target; **the main difficulty dial** (higher = harder).
- `n_targets` — set `>1` for multi-output regression.
- `bias`, `tail_strength`, `target_name`.

**Clustering** (`synthetic_generator_clustering.py`)
- `shape` — `"blobs"` (convex gaussian), `"moons"`, or `"circles"` (non-convex).
- `n_clusters`, `cluster_std` — number and spread of blobs; **higher std = more overlap = harder**.
- `center_box` — coordinate range for blob centers.
- `noise` — jitter for moons/circles.

## Tips & notes

- **Reproducibility:** the same `seed` yields the same DataFrame, including Faker columns. Change it to get a fresh draw, fix it for tests.
- **Don't leak the label.** In clustering, the `cluster` column is **ground truth for scoring only** (e.g. `adjusted_rand_score`) — drop it before fitting. Same idea for `target` in the supervised generators when you build feature matrices.
- **Faker columns are decoration by default.** Apart from the optional correlated `age`, the Faker fields are random and carry no signal — use them to test that your preprocessing handles strings, dates, high-cardinality categoricals, etc. Don't expect a model to "learn" from a random `email`.
- **Start easy, then harden.** Get your pipeline green on a clean, well-separated dataset first, then dial up `noise` / `cluster_std`, lower `class_sep`, and turn on `missing_frac` / `outlier_frac` / `duplicate_frac` to mimic messy reality.
- **Non-convex clustering:** `moons` and `circles` are 2D and will break centroid methods like k-means — useful for showing where DBSCAN / spectral clustering win.
- **Imbalance:** pair `weights` with metrics that tolerate it (precision/recall, PR-AUC) rather than plain accuracy.
- **Locale:** set `locale="pt_BR"` (or another) for region-appropriate names, addresses, and phone formats.

## Extending

- Add a Faker field by inserting it into the `field_makers` dict in any generator.
- To export to CSV: `df.to_csv("data.csv", index=False)`.
- For data that imitates a *real* dataset's distribution, these are the wrong tool — reach for SDV / CTGAN instead.
